# DESIGN.md — CV Verdict

## Design stance
CV Verdict should not look like a generic SaaS checker. It should feel like a serious evidence desk: document → proof → score → decision.

## Design constitution
Follow `docs/design/12_IMPECCABLE_FRONTEND_DESIGN.md`.

## Signature element
**Evidence Rail** — a vertical proof trail that appears across the analysis and results experience.

It should communicate that every score comes from visible evidence, not arbitrary AI judgment.

## Visual tone
- Serious.
- Clear.
- Sharp.
- Trustworthy.
- Not playful.
- Not corporate-template bland.

## Avoid
- Gradient text.
- Generic equal card grids.
- Decorative glassmorphism.
- Warm cream SaaS default.
- Big hero metric cliché.
- Over-rounded cards.
- Tiny uppercase eyebrows on every section.

## Interface copy
Use plain active language.

Arabic examples:
- قيّم السيرة.
- ارفع ملف PDF.
- احذف ملفي ونتيجتي.
- لم نستطع قراءة النص داخل الملف. ارفع نسخة PDF نصية.

English examples:
- Analyze CV.
- Upload PDF.
- Delete my file and result.
- No readable text was found. Upload a text-based PDF.

## Accessibility
- Strong contrast.
- Clear focus states.
- Keyboard navigation.
- Reduced motion support.
- Arabic RTL support.
- English LTR support.
